Countries Assignment: Condensed Workflow

High-value pre-exam guide | 5 phases, commit flow, pitfalls

Use This In Exam

  • Read once before coding; keep this open while solving.
  • Work phase by phase and commit only when phase criteria are done.
  • After each phase: compile, run, then commit.
Golden Rule: Do not skip verification steps. Most lost marks come from small unchecked mismatches.

5-Phase Workflow

Phase 1: Setup and Baseline

  • Initialize Git in the project root (same folder as pom.xml).
  • Set local Git identity (user.name, user.email).
  • Initial import commit.

Pitfalls: Wrong directory, missing full name, no initial commit.

Phase 2: Interface Class Skeleton

  • Create countries.CountriesInterfaceImpl.
  • Implement interface method stubs via IDE.
  • Add main and instantiate manager object.

Pitfalls: Wrong class name, wrong package, missing main.

Phase 3: Implement the 5 Stream Methods

  • Follow method Javadoc exactly.
  • Each method body should be stream pipeline only.
  • Call all 5 methods in main; print returned values.

Pitfalls: Extra logic outside pipeline, wrong collector, forgetting to print return values.

Phase 4: Branch + Dependency

  • Create feature branch (for example new-feature).
  • Add Gson dependency in pom.xml.
  • Sync Maven / run compile to confirm dependency resolves.

Pitfalls: Adding dependency on wrong branch, not syncing Maven, typo in XML tags.

Phase 5: New Feature + Final Merge

  • Add new interface method for JSON export.
  • Implement with Gson in CountriesInterfaceImpl.
  • Call in main, verify output, merge back to main.

Pitfalls: Interface and implementation mismatch, method not called, merge not completed.

Commit Map (Suggested)

git add . && git commit -m "initial import"
git add . && git commit -m "feat: add CountriesInterfaceImpl with stubs"
git add . && git commit -m "fix: implement 5 stream methods and main calls"
git checkout -b new-feature
git add . && git commit -m "build: add Gson dependency"
git add . && git commit -m "feat: add exportCountriesToJson feature"
git checkout main && git merge new-feature
Commit quality tip: One commit per meaningful phase is easier to debug and score.

High-Value Exam Tips

Before final submission:
  • Run compile once more from project root.
  • Open changed files and scan method signatures for exact matches.
  • Run main and check that output includes every required step.
  • Check branch state and merge status.
Mindset that wins marks: correctness first, then cleanliness, then speed.
